1. Escopo
COMO
Consultora de Projetos do Ciclo da Receita,
QUERO
disponibilizar uma API interna e um OPCODE customizado para listar faturas por
contexto de faturamento no Billing Care,
PARA
permitir que a tela Faturamento On Demand consulte faturas, apresente dados resumidos
e viabilize a seleção da fatura para a próxima ação operacional.

a API
TrialBillingInvoiceListAPI
e o OPCODE BRM
C_OP_LIST_INVOICE
.

Faturamento para cliente estrangeiro, que será tratado em US apartada.
•
Alteração no fluxo e processos AS IS do BRM.
•
Definição funcional, regras de navegação e UX/UI da tela Billing Care para faturamento on
demand.

4. Fluxo de Negócio
Fluxo Principal
O usuário autenticado acessa a tela Faturamento On Demand no Billing Care.
1.
O usuário informa filtros de consulta e o contexto de faturamento.
2.
A tela chama a API interna
TrialBillingInvoiceListAPI
.
3.
A API valida autenticação, autorização, filtros recebidos,

ordenação e
correlationId

4.
A API rejeita a consulta antes de acionar o OPCODE quando houver erro de entrada,
autenticação ou autorização.
5.
A API chama o OPCODE BRM
C_OP_LIST_INVOICE
quando a requisição estiver válida.
6.
O OPCODE consulta a origem de dados conforme o contexto informado.
7.
Quando
billingContext = TRIAL
, o OPCODE retorna somente registros com
trial_billing_status = 2
.
8.
Quando
billingContext = PRODUCTION
, o OPCODE retorna faturamentos FlexBill não
-
trial
conforme filtro de data.
9.
A API aplica paginação, ordenação, estrutura o payload de resposta e registra
rastreabilidade.
10.
A API retorna os dados resumidos compatíveis com a listagem da tela.
11.
A tela exibe as faturas disponíveis associadas ao cliente e ao contexto consultado.
12.
Após execução de Bill Now em fluxo apartado, a tela pode consultar novamente a API
para acompanhar status atualizado.
13.
